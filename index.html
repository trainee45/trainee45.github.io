    • A note about Tables:
      There are 3 classes of tables in DougieBase.
      1. The MAIN table is derived from each DougieBase database and is automatically generated when a new database is created. This is the table shown when you tap the DISPLAY TABLE btn on the Table Options information screen. A MAIN table can be edited, records and dynamic fields added or deleted, searched and sorted.
      2. The second class of table is one created from found records in a SEARCH of a MAIN database table. Data can be searched and sorted, but new records can not be added or deleted. If you need to ADD a record you have to revert back to the MAIN database table, from which it originated, and ADD/ or DELETE records to/from the MAIN table, and then repeat the SEARCH to recreate the changed table with the now added record.
      3. The third class of table references CONTACTS only, .. Contacts that you want to be associated with the database. CONTACTS tables can be created in 3 ways:
      A. By tapping the CONTACTS button in the menu of a MAIN database table - this method allows access to the CONTACTS app on your device.
      B. By tapping the CREATE TABLE button on the Table Information window that shows after tapping the TABLE button on the home screen main menu bar. The CREATE TABLE option has the same facility as the CONTACTS button in the Table window menu.
      C. By tapping the ADD CONTACTS/RECORDS button on the main database HOME screen menu. - This allows manual entry of Contacts data from a form sheet. There is no access to the ContactsPickerAPI (Contacts) on your device.
      CONTACTS tables generate 4 FIXED Fields - Name, DATE, Telephone, Email, and 1 DYNAMIC field .. Address ‚Ä¶ depending on your browser's capability to access the Address field from the ContactPicker API. The 3 types of Contacts tables allow editing of data in the table data cells, adding and deleting (Contacts) records,..(by tapping the + and - icons in the Table menu), and adding and deleting fields, by tapping the EDIT button in the Table menu. Editing of Field Names in the CONTACTS tables, by tapping the EDIT button in the Table menu, allows creation of CUSTOMIZED tables, thus facilitating their use for other purposes than just Contacts.
    • CUSTOM TABLES: Custom tables can be created in two ways:
      1.By using a CONTACTS table as a template, relabeling the FIELD names using the EDIT button, will create customized tables. The EDIT button allows renaming of FIELD headings, and ADDING or DELETING of fields, facilitating the creation of CUSTOM tables for any purpose.
      2. The ADD CONTACTS/RECORDS button in the Main menu allows labelling of the table field names on the fly, as you create the table and enter record data into the fields from a FORM input screen.
    • Field Labels:
      The FIELD LABELS button on the Table screen menu allows editing of the Contacts or new Table fields. CANCEL in this window resets fields to default values for a Contacts table. See documentation.
    • CONTACTS btn vrs Field Labels:
      To create contacts tables: First tap of CONTACTS btn creates a simple CONTACTS table using your device's ContactsAPI.
      A first tap of Field Labels Btn will create a CONTACTS table using your device's ContactsAPI but also adding an ADDRESS field.
      If you want a simple CONTACTS table with just 4 fields ..name,date,telephone,email..but want to edit the field labels..first tap CONTACTS to set up the 4 fields, and use CANCEL at the table naming stage..then return to the Field Labels btn, and use the EDIT CURRENT FIELDS option to get a simple CONTACTS table that you have been able to change the field names in.
      The CANCEL btn: This will reset contactsField names which will allow switching back and forth between 4 fields ending in EMAIL, OR 5 fields ending in ADDRESS. Depending on whether CONTACTS or FIELD LABELS is tapped first you get 4 or 5 fields. If you get 4 but you want 5 with ADDRESS just tap CANCEL in the Action Prompt window and/or in the ENTER NEW TABLE NAME window, and repeat the process following the prompts.
      Any table created using the ADD CONTACTS/RECORDS btn from the Main Menu will have a Fifth field initially labelled ADDRESS unless you edit the field names as per options provided.
    • WHEN ENTERING A LINK (To make it active) YOU MUST USE FORMAT: 'http://text of link label.com/'. Note the forward slash at the end of the link!
    • In FIXED field headers, (columns 1 to 4 of a table) use keywords TELEPHONE (3rd column header) EMAIL (4th column header) or in any DYNAMIC field header (columns 5 and up) which when double-tapped will activate telephone numbers to make phone calls or email addresses to create emails when the data cell in that column is also double-tapped.
    • Keywords TOTAL, AVERAGE, and COST or $ included in the field/column header of any DYNAMIC field header will perform the appropriate operation on that column of data when the field header is double-tapped. COST or $ can only be used with TOTAL or AVERAGE and simply adds a $ sign to the result.
    • Numerical data in data cells of DYNAMIC fields can be ADDED across the row leaving the sum total result in the last column of the row. If the field header of that row includes the keyword TOTAL, the last column can also be totaled vertically summing all the results of the record rows that have been summed across the row. You must adhere to this RULE for initiating summing values for numerical entries in the DYNAMIC fields of each row -
      RULE: 'sumAcross' keyword inserted in the data cell which is the last column of the FIRST record of the table and then subsequent records, will initiate summing of all numerical entries (in DYNAMIC fields only), across that record row of the table. The TOTAL keyword in the field heading of the last column in the table will facilitate adding all numerical entries vertically in the table column, IF THE LAST DATA CELL (last column) OF THE FIRST RECORD was initialized with the 'sumAcross' keyword. The TOTAL keyword in field headings of all other columns (DYNAMIC fields only) works as indicated."
    • If you have a column of data you do not want to be included in the summing across a record's fields, mark that number with an extra character, and it will be excluded from the sum. i.e 100 will be added in a sum across the records fields, but 100% or 100! Will NOT be included in the sum. However these numbers are included when summing or averaging numbers vertically in the column.
