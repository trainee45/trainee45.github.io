      The 3 ways to create a DougieBase table:

      There are 3 classes of tables in DougieBase.
      1. The MAIN table is derived from each DougieBase database and is automatically generated when a new database is created. This is the table shown when you tap the DISPLAY TABLE btn on the Table Options information screen. A MAIN table can be edited, records and dynamic fields added or deleted, searched and sorted.
      2. The second class of table is one created from found records in a SEARCH of a MAIN database table. Data can be searched and sorted, but new records can not be added or deleted. If you need to ADD a record you have to revert back to the MAIN database table, from which it originated, and ADD/ or DELETE records to/from the MAIN table, and then repeat the SEARCH to recreate the changed table with the now added record.
      3. The third class of table references CONTACTS only, .. Contacts that you want to be associated with the database. CONTACTS tables can be created in 3 ways:
      A. By tapping the CONTACTS button in the menu of a MAIN database table - this method allows access to the CONTACTS app on your device.
      B. By tapping the CREATE TABLE button on the Table Information window that shows after tapping the TABLE button on the home screen main menu bar. The CREATE TABLE option has the same facility as the CONTACTS button in the Table window menu.
      C. By tapping the ADD CONTACTS/RECORDS button on the main database HOME screen menu. - This allows manual entry of Contacts data from a form sheet. There is no access to the ContactsPickerAPI (Contacts) on your device.
      CONTACTS tables generate 4 FIXED Fields - Name, DATE, Telephone, Email, and 1 DYNAMIC field .. Address ‚Ä¶ depending on your browser's capability to access the Address field from the ContactPicker API. The 3 types of Contacts tables allow editing of data in the table data cells, adding and deleting (Contacts) records,..(by tapping the + and - icons in the Table menu), and adding and deleting fields, by tapping the EDIT button in the Table menu. Editing of Field Names in the CONTACTS tables, by tapping the EDIT button in the Table menu, allows creation of CUSTOMIZED tables, thus facilitating their use for other purposes than just Contacts.
    • CUSTOM TABLES: Custom tables can be created in two ways:
      1.By using a CONTACTS table as a template, relabeling the FIELD names using the EDIT button, will create customized tables. The EDIT button allows renaming of FIELD headings, and ADDING or DELETING of fields, facilitating the creation of CUSTOM tables for any purpose.
      2. The ADD CONTACTS/RECORDS button in the Main menu allows labelling of the table field names on the fly, as you create the table and enter record data into the fields from a FORM input screen.

The TOTAL keyword included in field headings of all other columns (DYNAMIC fields only) works as indicated."

    I deleted a database, but a notiÔ¨Åcaton appears saying unable to DELETE ..Ô¨Åle is LOCKED by browser. Now what?

    If you have viewed or otherwise acted on a database you subsequently DELETE, you will find the browser has put a temporary LOCK on the file. If DougieBase initiated the DELETE such as occurs during a RENAME process or during a creation of a new budget sheet, DougieBase waits for the 'time out' and then will DELETE the file. You will receive a notification when this occurs. If the user DELETED the file, wait for the browser to release its LOCK and try again. If you close the app and restart the app this sometimes releases the LOCK. In any case DougieBase remembers that a lock was placed on the file and on restart highlights the filename in green with a red border immediately on re-listing the file names, if it's a budget Template file, and in grey with red border (after attempting to re-open), if it's a previously deleted database, reminding the user to DELETE the file again. Go direct to MANAGE, tap on the appropriate file to be deleted and tap DELETE. Don't open the file before doing this as the browser will then put another LOCK on it.

    I uploaded an updated version of a database but nothing happened?

    To view an updated version of a pre-existing database that is currently listed in your current database filenames window, you must first delete the old version from DougieBase. (Make sure the new version is available and backed up). DougieBase will not load in two instances of a database with the same name! If you have viewed the older version, and then try to delete the older version so as to update it to the more recent version, the browser will have put a temporary LOCK on the file, and it will then be listed as 'BLOCKED'. In this scenario you have to wait until DougieBase releases its lock and deletes the file before you can RESTORE or load in the newer version.
    If you want to keep the older version as well, then RENAME it with an appropriate filename, prior to deleting it.

    In a budget sheet template for any month, the DATE field is the same for all items, around Nov 2022? Why?

    In a regular DougieBase database, the DATE field reflects the date any particular record in the database was created or added to the database. Monthly Budget Sheets are generated by the normal DougieBase database engine, and thus the DATE field in a budget sheet template is less functional. The date reflects when the budget sheet template itself was created. If you add another record or item to your budget sheet, the DATE field for that item will reflect the date on which you added the item.

    If I edit a database, and then back it up using the same 'filename', I get an error message saying 'filename.txt' already exists in this location. Can not rename 'filename'? Do I have to use a modified filename or a different location to save the backup?

    The DougieBase SAVE to FILES + (delete) shortcut has been reprogrammed to take this into account. The shortcut now automatically appends the timestamp to the filename, thus altering the name to a unique filename allowing the shortcut to make the SAVE. This means you will have more than one iteration of 'filename.txt' but each with a different timestamp. You can now choose to clean up your DougieBase storage folder, and delete any 'filename.txt + timestamp' that you do not want to keep..

    How do I best add an image to a database in a specified IMAGE field?

    Not all public photo storage web sites for security reasons will allow a third party app to download and view photos. Designate a field in your database to hold links to images that you have stored in the cloud. Three of the best servers for this are Dropbox and One Notes One Drive, and Box. Google Drive, Amazon Photos and Apple iCloud Drive also will work for this. Create a field in your database and label it 'IMAGE'. Go to your photo cloud storage and obtain the link to your photo. (The link will be 'copied' to your clipboard.) In One Drive you need to use the share sheet option and email yourself the link, and 'copy' it from your email. Now 'paste' just the link (no extra words or comments) into the appropriate td data cell via the standard DougieBase EDIT process. Save the table. Now you can view the photo simply by tapping on the link. NOTE: You can put just about any link to any web page into the 'IMAGE' field data cell and it will work. The reverse is not true. Putting an IMAGE link into any field data cell not labelled 'IMAGE' will cause an error on loading the image. Also note that the field heading label needs only contain 'IMAGE'. So both 'IMAGE' and 'Locomotive IMAGE' field labels will work.
